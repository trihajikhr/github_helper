# About GitHub and Git

## Tentang GitHub

GitHub adalah platform berbasis _cloud_ tempat Anda dapat menyimpan, berbagi, dan bekerja sama dengan orang lain untuk menulis kode.

Menyimpan kode Anda dalam sebuah "repository" (repositori) di GitHub memungkinkan Anda untuk:

- **Memamerkan atau berbagi** pekerjaan Anda.
- **Melacak dan mengelola perubahan** pada kode Anda dari waktu ke waktu.
- Mengizinkan orang lain **meninjau** kode Anda, dan memberikan saran untuk memperbaikinya.
- **Berkolaborasi** dalam sebuah proyek bersama, tanpa khawatir bahwa perubahan Anda akan memengaruhi pekerjaan kolaborator Anda sebelum Anda siap untuk mengintegrasikannya.

Kerja kolaboratif, yang merupakan salah satu fitur mendasar GitHub, dimungkinkan oleh perangkat lunak sumber terbuka (open-source), Git, yang menjadi dasar dibangunnya GitHub.



## Tentang Git

Git adalah sistem kontrol versi (*version control system*) yang secara cerdas melacak perubahan dalam berkas (*files*). Git sangat berguna ketika Anda dan sekelompok orang semuanya melakukan perubahan pada berkas yang sama di saat yang bersamaan.

Biasanya, untuk melakukan ini dalam alur kerja berbasis Git, Anda akan:

* **Membuat cabang** (*branch*) dari salinan utama berkas yang sedang Anda (dan kolaborator Anda) kerjakan.
* **Melakukan pengeditan** pada berkas secara independen dan aman di cabang pribadi Anda sendiri.
* Membiarkan Git secara cerdas **menggabungkan** (*merge*) perubahan spesifik Anda kembali ke dalam salinan utama berkas, sehingga perubahan Anda tidak memengaruhi pembaruan orang lain.
* Membiarkan Git **mencatat** (*keep track*) perubahan Anda dan perubahan orang lain, sehingga Anda semua tetap bekerja pada versi proyek yang paling mutakhir.

Untuk mencoba menggunakan Git sendiri, lihat [Getting started with Git](https://docs.github.com/en/get-started/learning-to-code/getting-started-with-git).
